User Roles and Workflow Views

The assignment of a user type depends on your duties and role within the care management team. Persivia’s Chronic Care Management (CCM) system provides the following five user roles, each bringing a corresponding set of privileges.

  • Super Administrator: Users having this role and status can add other authorized users to the pool of active users. Super Administrator users can also edit user profiles; they have corresponding access to the User Management tab. Only a Super Administrator can create or authorize another Super Administrator.  However, users in this group do not have access to patient information.

  • Care Manager: The Care Manager can view, add, and edit patient information. Care Managers can access all pages and tabs on the portal (except the Change Log that lists Care Plan approval events). Care Managers need not be doctors, and one of the benefits of the Care Management Suite is to enable non-physicians to create care solutions (such as Care Plans) in draft, that physicians can review and selectively approve.

  • Care Physician: Care Physicians can edit patient information and access all pages and tabs on the portal, including the Change Log. This is the only role that will be allowed to approve care plans.

  • Care Supervisor: This User will be able to edit patient information and access all pages and tabs on the portal except the User Management and the System Log tabs. The Care Supervisor privileges are like those of a Care Manager.

  • Reviewer: This User will have read-only privileges to the patient data and will not be permitted to make any modifications or deletions. Reviewers access the User Management and System Log tabs.
